proxy_ssl certificates in nginx

Client side certificates allow the server to authieticate the source of the request. The server gives you a key file and a certificate that you have to send along with a request for it to be accepted. Nginx can not only serve requests but also act as a proxy between requests to to other servers. Hence the name reverse proxy server. Nginx's directive allow you to configure both server side and client side certificates. We'll see how to do both but first let us clarify the terminology a bit.

Server side certificate: By server side certificate, in this context, we mean that the downstream requests that is the requests coming in to nginx must be over a secure connection.

Client side certificate: By client side certificate, in this context, we mean that all the upstream requests, that is the requests going out of nginx are over a secure connection. A better term would be proxy certificates.

Server side certificates are configured by ngx_http_module
