proxy_ssl certificates in nginx

Client side certificates allow the server to authenticate the source of the request. The server gives you a key file and a certificate that you have to send along with a request for it to be accepted. Nginx can not only serve requests but also act as a proxy between requests to to other servers. Hence the name reverse proxy server. Nginx's directive allow you to configure both server side and client side certificates. We'll see how to do both but first let us clarify the terminology a bit.

Server side certificate: By server side certificate, in this context, we mean that the downstream requests that is the requests coming in to nginx must be over a secure connection.

Client side certificate: By client side certificate, in this context, we mean that all the upstream requests, that is the requests going out of nginx are authenticated. A better term would be proxy certificates.

Server side certificates are configured by ngx_http_module where as the proxy certificates are handled by the directives in the ngx_http_proxy module.

So how do the client side certificates work?

Essentially the purpose of client side certificates is authentication of the client to the server. The client sends it's certificate to the server. The server checks whether the client certificate is valid. If it is the connection is established for future requests and if not the request for connection is denied.

Where are they used?

Since the responsibility of managing client side certificates is on the client, they are generally used in scenarios where determining the identity of the client becomes essential. Consider that you have an api that you want to expose to a set of valid users. In that case you can issue each user a certificate and keep a copy for yourself. Only when a user sends the right certificate you allow access to the api calls otherwise just send an unauthorized response.

Is there any benefit in letting nginx handle the certificates?

Yes. For one ssl handling in nginx is extremely optimized. It's fast and light. And second it's easy. All you got to do is add few directives and you are set.
